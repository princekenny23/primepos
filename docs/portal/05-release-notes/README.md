# Release Notes Standard

## Purpose
Provide clear updates for users, support teams, and partners.

## Format
- Version: `vX.Y.Z`
- Date: `YYYY-MM-DD`
- Sections:
  - New Features
  - Improvements
  - Bug Fixes
  - Notes for Operators

## Writing Rules
- Keep each bullet short and user-focused.
- Avoid internal-only jargon.
- Mention impact and required action if any.

## Template
```md
# Release vX.Y.Z - YYYY-MM-DD

## New Features
- Added ...

## Improvements
- Improved ...

## Bug Fixes
- Fixed ...

## Notes for Operators
- Action required: ...
```

## Storage Convention
- One file per month for lightweight teams, example `2026-04.md`
- One file per release for larger teams
