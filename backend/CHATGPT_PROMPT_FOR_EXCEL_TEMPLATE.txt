Create an Excel template file (.xlsx) for bulk product import with the following specifications:

FILE REQUIREMENTS:
- Format: Microsoft Excel (.xlsx)
- File Name: product_import_template.xlsx
- Sheet Name: "Products"
- Encoding: UTF-8
- Architecture: UNITS ONLY (No variations/ItemVariation model)

COLUMN STRUCTURE (14 columns in this exact order):
1. Product Name (Required, Text, max 255 chars) - Example: "Coca Cola 500ml"
2. Retail Price (Required, Number, 2 decimals, min 0.01) - Example: 1500.00
3. Unit Name (Optional, Text, default "pcs") - Example: "bottle", "case", "kg"
4. Conversion Factor (Optional, Number, decimal, min 1.0) - Example: 12 (for case of 12 bottles)
5. Wholesale Price (Optional, Number, 2 decimals) - Example: 1200.00
6. SKU (Optional, Text, max 100 chars) - Leave empty for auto-generation
7. Category (Optional, Text, max 255 chars) - Example: "Beverages"
8. Barcode (Optional, Text, max 100 chars) - Example: "1234567890123"
9. Cost (Optional, Number, 2 decimals, min 0) - Example: 1000.00
10. Description (Optional, Text) - Example: "Carbonated soft drink"
11. Low Stock Threshold (Optional, Integer, default 0) - Example: 10
12. Initial Stock Qty (Optional, Integer, default 0) - Initial batch quantity to receive
13. Batch Expiry Date (Optional, Date, format YYYY-MM-DD) - Example: "2026-08-04"
14. Is Active (Optional, Text, default "Yes") - Valid values: "Yes" or "No"

HEADER ROW (Row 1):
- Column headers: Product Name | Retail Price | Unit Name | Conversion Factor | Wholesale Price | SKU | Category | Barcode | Cost | Description | Low Stock Threshold | Initial Stock Qty | Batch Expiry Date | Is Active
- Format: Bold text, light blue background (#D9E1F2), centered, with borders
- Font size: 11pt

FORMATTING REQUIREMENTS:
- Header row: Bold, light blue background (#D9E1F2), centered, borders on all cells
- Data rows: Regular text, white background, borders on all cells
- Column widths: Product Name (25), Retail Price (12), Unit Name (12), Conversion Factor (16), Wholesale Price (14), SKU (15), Category (20), Barcode (15), Cost (12), Description (40), Low Stock Threshold (18), Initial Stock Qty (14), Batch Expiry Date (15), Is Active (12)
- Number formatting: 
  * Retail Price, Wholesale Price, Cost: 2 decimal places
  * Conversion Factor: 1-2 decimal places (allow 1.0, 12.0, 0.5)
  * Stock/Threshold: Whole numbers (integers)
  * Batch Expiry Date: YYYY-MM-DD format
- Freeze header row (Row 1)

DATA VALIDATION:
- Retail Price: Decimal, minimum 0.01
- Wholesale Price: Decimal, minimum 0 (optional)
- Conversion Factor: Decimal, minimum 0.01, maximum 999.99
- Initial Stock Qty: Whole number, minimum 0
- Low Stock Threshold: Whole number, minimum 0
- Unit Name: List (pcs, bottle, case, kg, g, l, ml, box, pack, unit, each)
- Batch Expiry Date: Date format YYYY-MM-DD (optional, future dates only)
- Is Active: List (Yes, No)

IMPORTANT NOTES - UNITS ONLY ARCHITECTURE:
- NO VARIATION_NAME FIELD (ItemVariation model removed in migration 0016)
- Each product can have MULTIPLE UNITS (e.g., bottle, case) via Unit Name + Conversion Factor
- Unit Name: The unit of measurement for THIS row's product entry (bottle, kg, case, etc.)
- Conversion Factor: How many base units = this unit (e.g., 12 bottles = 1 case)
- If importing same product with different units, create separate rows with same Product Name
- Batch Expiry Date creates a batch record automatically on import
- If Batch Expiry Date is empty, batch will have no expiry (WARNING: stock never expires)
- Categories will be auto-created if they don't exist
- Column names are case-insensitive but use exact names for headers
- First row must be headers
- Maximum 1000 rows per import
- SKU column should be empty in sample rows (for auto-generation)

EXAMPLE: Multi-unit Product (Coca Cola)
- Row 2: Unit="bottle", Conversion=1, Price=1500 (retail per bottle)
- Row 3: Unit="case", Conversion=12, Price=15000 (retail per case of 12)
- System will create: Product "Coca Cola 500ml" with ProductUnit records for bottle and case
- Creating a sale with "case" unit will deduct 12x the base quantity
